Test Case: Validation of the form with "@Eager" fields.

When change of the "@Eager" field is posted to the server, only changed field should be validated. All fields are validated by "@Size(max=5)" annotation. Field "Text Eager" has additional "validate" Method. Inpit "qwe" is considered as wrong input and causes additional validation violation.

Test scenario:

  1. Focus "Text Eager" field. Write input "abc". Leave the field.
  2. Field "Text Eager Echo" equals "Text Eager: abc". No validation errors.
  3. Focus "Text Eager" field. Write input "abcdef". Leave the field.
  4. Field "Text Eager Echo" remains unchanged. Validation error for "Text Eager" is displayed.
